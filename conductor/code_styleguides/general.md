# JavaScript Style Guide

1.  **Formatting:** Use 4 spaces for indentation. Semicolons are required.
2.  **Naming:** `camelCase` for variables and functions. `PascalCase` for classes.
3.  **Modern JS:** Use `const` and `let`. Avoid `var`. Use arrow functions `() => {}` where appropriate.
4.  **Async/Await:** Prefer `async/await` over raw Promises for readability.
5.  **Comments:** Document complex logic.

# Python Style Guide

1.  **PEP 8:** Adhere to standard PEP 8 guidelines.
2.  **Formatting:** 4 spaces per indentation level.
3.  **Type Hinting:** Use type hints (e.g., `def my_func(a: int) -> str:`) where helpful.
4.  **Docstrings:** Use docstrings for all modules, classes, and functions.
